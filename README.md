# Fraud Detection Pipeline — End-to-End Supervised Learning Project

A complete, production-style fraud detection pipeline: EDA → cleaning → feature engineering →
stratified split → SMOTE (train-only) → 6 models → hyperparameter tuning → imbalance-aware
evaluation → model comparison → feature importance → persistence → inference on new data.

## ⚠️ About the dataset
This environment has no general internet access, so the classic Kaggle `creditcard.csv` could not
be downloaded. `notebooks/fraud_detection_pipeline.ipynb` instead **generates a synthetic
transaction dataset** (50,000 rows, ~0.6% fraud, with injected missing values and duplicate rows)
that mirrors the real dataset's structure — anonymized PCA-like `V1`–`V27` features, `Time`,
`Amount`, and a binary `Class` target. Every technique in the notebook is dataset-agnostic: to use
a real dataset, only the Step 2 loading cell needs to change.

## Results summary (test set, ranked by PR-AUC)

| Rank | Model | Precision | Recall | F1 | ROC AUC | PR AUC |
|---|---|---|---|---|---|---|
| 1 | **SVM (RBF)** | 0.667 | 0.492 | 0.566 | 0.834 | **0.501** |
| 2 | XGBoost | 0.605 | 0.354 | 0.447 | 0.930 | 0.453 |
| 3 | Random Forest | 0.226 | 0.292 | 0.255 | 0.924 | 0.201 |
| 4 | KNN | 0.096 | 0.754 | 0.170 | 0.881 | 0.200 |
| 5 | Logistic Regression | 0.024 | 0.739 | 0.046 | 0.847 | 0.156 |
| 6 | Decision Tree | 0.027 | 0.585 | 0.051 | 0.794 | 0.047 |

**Why PR-AUC, not accuracy, drives model selection:** with ~0.6% fraud, a model predicting
"legitimate" for every transaction scores >99% accuracy while catching zero fraud. PR-AUC
(Average Precision) ignores the huge, trivial true-negative count and focuses on how well the
model finds the rare fraud class — see the notebook's Step 12 for the full explanation.

**Selected model: SVM (RBF kernel)** — highest PR-AUC and F1, and the best precision/recall
balance of the six candidates. XGBoost is a strong, much faster runner-up (0.4s train vs. SVM's
137s) worth considering if inference/retraining latency matters more than the last few points of
PR-AUC in a given deployment.
